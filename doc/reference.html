Component parameters

Components can be parametrized in different ways. By parameter, we mean properties of its controllers, which can be set in the following manner:

  1. in the init method of the controller;
  2. as parameters in the e and f namespaces of the controller element (when instatiating a controller) see the timer example;
  3. as parameters in the e and f namespaces of the component element (when instantiating a component.)

Initializations are done in that order so that values can be successively overridden. A component view can then then reference parameters; attribute strings of the form {x} which, at instantiation time, get replaced with the value of the property x of the default controller of the component.

When a template value is used but there is no corresponding value, the attribute is ignored. This is useful for optional attributes, when no default value is provided.

In the example below, a component called circle is defined. Its parameters (cx, cy, r, fill and opacity) define how it is rendered as an SVG circle. These parameters are set randomly in the init method of the default controller. Further down in the application, three instances are created; the first one has two default parameters setting the center of the first circle to the center of the view box, while the third one has a stroke attribute which is only rendered for that particular instance since the other two have no stroke property.

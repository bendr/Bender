Examples

Component rendering

  • Create a context tree, and that's it (no output.)
  • Hello, world! — build a minimal Bender tree that displays a message.
  • Hello, world!/text only — build a minimal Bender component that displays a text-only message.
  • Hello, world!/use — load a minimal Bender component that displays a message.
  • Multiple contexts — create multiple Bender contexts, each with their own component, in the same host document.
  • Simple content — simple test of content and default content.
  • Use several components — use several different components in the same component.
  • Target element — rendered once and preserving the id attribute in "unique" elements; the gradient is thus accessible through its id.

Tree modification

  • Update instances of a component after its view content has changed.

Properties

  • Property in attribute — the square attributes (position, size, color) are set using properties.
  • Show property — show the value of a property in a text node; try setting u._instance.properties.foo in the console and see the live updates.
  • Show computed property value — show a computed property in a text node
  • Chain properties — y depends on x; z depends on both x and y; y depends on z.

Error handling

  • Error loading an unavailable component.

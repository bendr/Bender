Dumber

Dumber is a simpler implementation of Bender intended to prototype dynamic rendering of components.

Manual

Getting started

Using Dumber just requires including the flexo.js (the support library for Dumber) and dumber.js Javascript files in the host document (e.g., in script tags.) See for example the above test documents.

The context

The context is the document hosting Dumber nodes. The root of the document is a context element, which should only be used as the root a Dumber tree. dumber.create_context(target) creates a new context with a rendering target specified by target (or the current document by default.) If the target is a DOM Element (e.g. the body of an HTML document, or a g element in an SVG document), rendered nodes will be appended to the given target. In other cases (e.g., when the target is a Document node, which is the default), individual components may be rendered by specifying a target node. The context root element is returned (not the context document itself, which can be accessed through the DOM ownerDocument property of any node in the document.)

Dumber nodes

Nodes in the Dumber tree overload common DOM methods so that changes can be tracked and reflected after rendering was done. Component elements, such as the context root element, have a convenience method $(name, attributes, ...) to create new nodes.

  • The name parameter may be qualified with a known namespace prefix (html or xhtml, svg, xlink, xml, xmlns; the Dumber namespace is used if there is no prefix) and may include an id prefixed by # and a list of class names prefixed by .. For example, the name string html:div#body.calculator describes an HTML div element with id body and class calculator, while use describes a Dumber use element. Note: id must come before classes.
  • The attributes parameters is an object which keys are attribute names and values attribute values. For example, { q: "app" } sets the q attribute to app for the node being created. A null or undefined value for an attribute will not set it. This parameter can be omitted.
  • The following parameters (if any) are the contents of the node. Valid content is either a string, which creates a text node, and other nodes, usually created themselves through the $ method.

Node manipulation is done through the DOM methods appendChild, insertBefore, setAttribute, and setAttributeNS. Additionally, Dumber nodes have a _textContent(text) method to replace setting the textContent property, which can still be read normally (and also be set, but changes will not cause the rendering to update.) Dumber nodes have an internal method _refresh(parent) which will refresh the rendering for the parent of this node (which defaults to parentNode if not specified; the parameter is needed mostly for updates after a node has been removed from its parent.) It should not be necessary to call this method under normal circumstances.

Since a Dumber tree may be modified at runtime, it can be serialized again through the _serialize() method. This simply returns a text string of the XML serialization of the current state of the Dumber tree. This method is available on any node, so it may be used for instance for the main application or a given component rather than the full context.

The component, app and context nodes

The component node describes a Dumber component. app is synonymous with component and is provided to describe the main component of a Dumber application. context is also a synonym and is used as the root of the Dumber tree (and as such should not be used elsewhere.) A component may have metadata, a view, scripts, watches, and sub-components. These are described by the following nodes:

  • desc contains textual description for the component. This is used as metadata; for intance, it can be used to give hints about the usage of this component.
  • script allows customizing the behavior of a component through Javascript.
  • title contains the title of the component as plain text. This is used mostly as metadata, but gets rendered as the document title when it is the child of an app element which in turn is the child of a context root element.
  • use creates a new instance of a component and renders it in this component's view (including in a context node; see below.) use nodes should appear as descendants of the view node of the component.
  • view contains foreign nodes that will be rendered in the target document when this component is rendered. A component may have only one view; setting a new view automatically replaces the previous one.
  • watch describes the behavior of the component in the face of changes in its properties and events it receives. See the section on watches below.

All Dumber nodes may have an id attribute which should be unique in the document where the node appears (if importing nodes from different documents, nodes with the same id can be distinguished by the URL of the original document.) These ids are not rendered since many instances of the same component may be rendered in a document.

The use node

The use node instantiates and renders a component. For every use node, a new instance of a component is created; if this component has a view, or has use children that themselves have a view (at any level of nesting) then the contents of the view gets rendered in its place.

The component to be instantiated is designated by one of the following attributes:

  • q (short for query selector) to refer to a node in the current context (TODO check the exact context of the querySelector call); note that using # for id may not work, so it is safer to use ref below in that situation;
  • ref to refer to the id of a node in the current document;

Additional attributes (besides id) and contents of the use nodes are passed to the instantiated component view during rendering as described below.

The view and content nodes

The view node contains foreign nodes (e.g., HTML, SVG, etc.) describing the actual rendering of a component. view may also include use and content nodes (other Dumber nodes are silently ignored), as well as text nodes. Additionally, use nodes outside of a view node are also rendered as siblings of the children of the view node.

When a component is rendered, the contents of the view are copied as is (minus id attributes as discussed above) into the target element. use nodes allow to include Dumber components inside another component. The content allows to customize the rendering of a node by replacing this node with the contents of the original use. The content node may have content of its own, which is the default content that gets rendered when the use has no content.

Similarly, attributes of the use node are added to the top-level elements that get rendered; this allows for instance to change the class of an element to alter its rendering.

The component instance object and the script node

Before rendering, a component gets instantiated, which creates a new Javascript object acting as the component instance of this component. A component instance has the following methods and properties:

  • init(use, component) initializes the component after it was created. use is the use node instantiating the component, and component is the component node being instantiated. The instance is returned.
  • views maps the id of the foreign nodes inside the view of the component and the actual rendered nodes in the target. Nodes that do not have an id are not included.
  • uses maps the id of use nodes inside the view of the component and the instance that was created. Nodes that do not have an id are not included.
  • instantiated() is called after the instance was created and initialized, and right before it will be rendered. By default, it does not do anything but can be overloaded for component customization.

The script node contains custom Javascript code. This code is executed when the node is instantiated defines a variable prototype to access the prototype of the component that it is part of. It is therefore possible to overload the base component's methods as well as add new methods to that component. A good entry point for initialization is the instantiated method.

Properties: instances have properties (Javascript values) that can be set directly from Dumber attributes. Attributes in the namespace http://dumber.igel.co.jp/p or prefixed with p: (when using the $ convenience function to create nodes) on component and use nodes sets the value of a property with the same name as the attribute. When set on a component node, all instances will have this property with the value as default; when set on a use node, then only that instance will have that property (or value if the component also has that property.) Instances have a properties field that holds the properties.

The watch, get and set nodes

Component and watch instances

Before a component can be rendered, it must be instantiated. Many instances of the same component may be created and rendered independently, maintaining their own state.

Rendering a Dumber tree

A self-contained Dumber application tree can be rendered into any host document by simply creating a context with the host document as target, populating the tree, and having a use element as a child of the root element pointing to the main application component. Here is a simple HTML "Hello, world!" example being rendered in the body of the host document:

var context = dumber.create_context(document.body);
context.appendChild(
  context.$("component",
    context.$("view",
      context.$("html:p", "Hello, world!"))));
context.appendChild(context.$("use", { q: "component" }));

Rendering a Dumber component in a document

Another use case is to render individual components into an existing document. The difference is that we render directly a component element into the given target, using the _render_in(target) where target is the parent DOM node where the content will be rendered. For example, to render a component in a div element with the id here:

var context = dumber.create_context();
context.appendChild(
  context.$("component",
    context.$("view",
      context.$("html:p.dumber", "But this is a Dumber component!"))));
context.querySelector("component")
  ._render_in(document.getElementById("here"));
